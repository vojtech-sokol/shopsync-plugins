# Money S3 â€” XML format reference

Money S3 (classic, non-API) reads and writes XML files in a fixed envelope. Every document exchanged with Money is wrapped in `<MoneyData>` and dispatched via `Money S3.exe /XML` (see [cli-reference.md](cli-reference.md)).

## Envelope

```xml
<?xml version="1.0" encoding="windows-1250"?>
<MoneyData
    description="free text"
    ExpZkratka="_O+P+N"
    ExpDate="2026-04-19"
    ExpTime="10:23:00"
    VyberZaznamu="0">
  <SeznamObjPrij>
    ...
  </SeznamObjPrij>
</MoneyData>
```

- Exports produced by Money (and by E-shop konektor) are `encoding="windows-1250"`.
- Imports generated by our side use `encoding="utf-8"` and Money re-encodes them. The PHP templates under `lib/moneys3/templates/*.php` write utf-8 literally and use the `xmlStr()` helper for escaping.
- Root element of every list is `Seznam<Entity>` (e.g. `SeznamZasoba`, `SeznamObjPrij`, `SeznamFaktVyd`, `SeznamFirem`, `SeznamKmKarta`).

## Key elements per entity

### Stock card (`_ZAS`, `Zasoba` + `KmKarta`)

```xml
<Zasoba>
  <konfigurace>
    <Druh_zas>Z</Druh_zas>
    <SDPH_Nakup>21</SDPH_Nakup>
    <SDPH_Prod>21</SDPH_Prod>
  </konfigurace>
  <StavZasoby>
    <Zasoba>0</Zasoba>
    <Rezervace>0</Rezervace>
    <Objednano>0</Objednano>
  </StavZasoby>
  <KmKarta>
    <Popis>...</Popis>
    <Zkrat>...</Zkrat>
    <Katalog>...</Katalog>
    <UzivCode>...</UzivCode>
    <BarCode>...</BarCode>
    <GUID>{...}</GUID>
    <MJ>ks</MJ>
    <Hmotnost>0</Hmotnost>
    <TypKarty>jednoducha</TypKarty>
  </KmKarta>
  <Sklad>
    <Nazev>HL</Nazev>
    <KodSkladu>HL</KodSkladu>
    <GUID>{...}</GUID>
  </Sklad>
  <Skupina>
    <Zkratka>...</Zkratka>
    <Nazev>...</Nazev>
  </Skupina>
  <PC>
    <SDPH>1</SDPH>
    <Hladina><Zkrat>MO</Zkrat></Hladina>
    <Cena1><Cena>190</Cena><Sleva>0</Sleva></Cena1>
  </PC>
  <Nak_Cena>...</Nak_Cena>
  <WWWPopis>...</WWWPopis>
  <WWWPopis2>...</WWWPopis2>
  <Vyrobce>...</Vyrobce>
  <eshop>
    <IN_Export>6</IN_Export>
    <CisKarty>99</CisKarty>
    <eSkup>...</eSkup>
  </eshop>
</Zasoba>
```

Field notes:
- `Druh_zas`: `Z` = stock, `O` = commercial, `V` = production, `S` = service.
- `SDPH_Nakup` / `SDPH_Prod` carry the VAT rate as a **percent**, not x100.
- `PC/SDPH`: `1` = price in `Cena1/Cena` includes VAT, `0` = net. `loadPrices()` in `lib/moneys3/products.php` reads this flag.
- `Hladina/Zkrat` is the price-level shortcut (cenĂ­k / cenovĂ¡ hladina). The loader matches it against `set_pricegroup1..6` constants.
- `eshop` block is only populated when E-shop konektor is active; see [konektor-vs-classic.md](konektor-vs-classic.md).

### Issued invoice (`_FP+FV`, `FaktVyd`)

Same partner/item shape as `ObjPrij` plus:

```xml
<FaktVyd>
  <GUID>{...}</GUID>
  <Doklad>FV2025-001</Doklad>
  <ZkratkaTyp>FV</ZkratkaTyp>
  <Typ>FV</Typ>
  <DatUcPr>...</DatUcPr>
  <PlnenoDPH>...</PlnenoDPH>
  <Splatno>...</Splatno>
  <KodDPH>22Tuz</KodDPH>
  <Ucet>BAN</Ucet>
  <PredKontac>F001</PredKontac>
  <VarSymbol>...</VarSymbol>
  <eshop><IN_Export>0</IN_Export></eshop>
  <SeznamPolozek>
    <Polozka>...
      <SklPolozka>
        <KmKarta>...</KmKarta>
        <Sklad>...</Sklad>
      </SklPolozka>
    </Polozka>
  </SeznamPolozek>
</FaktVyd>
```

- Unlike `ObjPrij`, invoices wrap `KmKarta+Sklad` inside `<SklPolozka>`.
- `KodDPH` is set from `getCfg(1,"typ_dph")`; becomes `OSS` when OSS routing kicks in.
- `ZkratkaTyp` / `Typ` come from `getCfg(1,"typ_skladoveho_dokladu")` â€” omit the block when empty.

## PHP encoding helpers

`lib/functions.php` supplies these â€” they are used in every template in `lib/moneys3/templates/`:

- `xmlStr($s, $max=null)` â€” escapes `& < > " '`, trims, optionally truncates. Use around every inserted value.
- `create_guid($seed)` â€” deterministic GUID from an arbitrary seed; wrap in `{...}` when Money requires braces (most elements do).
- `md53($s)` â€” short hash used for `KodPartn` (10-char partner code) and similar fixed-width fields.
- `maxLen($s, $n)` â€” truncate string to `$n` chars.
- `sqlSafe($s)` â€” escape single quotes for inline SQLite sql (used when we populate `sync_storecards`).

## OSS (one-stop-shop) branching

Templates use a shared switch: if `getCfg(1,"oss")==1` AND delivery country != `getCfg(1,"oss_hlavni_zeme")` AND we know that country's VAT rates in `$GLOBALS["eu_vat_rates"]`:

- Orders set `<StatMOSS>`, `<ZpVypDPH>1</ZpVypDPH>`, `<SazbaDPH1>` (low) and `<SazbaDPH2>` (high).
- Line items override `<SazbaDPH>` with the country's high rate.
- Invoices additionally use `<KodDPH>OSS</KodDPH>` instead of `getCfg(1,"typ_dph")`.

`$GLOBALS["eu_vat_rates"]` is loaded in `lib/moneys3/inc.php` from `https://shopsync.cz/vatrates_s3.json`.

## VAT conventions

- `<SazbaDPH>` on line items is **percentage x 100** (e.g. 21% -> `2100`). Matches the `$item["vat"] * 100` pattern in templates.
- `<SDPH>` inside `<PC>` is a boolean flag: `1` = stored price includes VAT, `0` = stored price is net. On read, `lib/moneys3/products.php::loadPrices()` uses it to compute `price` vs `price_novat`.

## Currency

Home-currency templates emit `<Celkem>` and `<Cena>` in the home currency. Foreign currency triggers a `<Valuty>` block:

```xml
<Valuty>
  <Mena><Kod>EUR</Kod><Mnozstvi>1</Mnozstvi><Kurs>25.1</Kurs></Mena>
  <Celkem>...</Celkem>
</Valuty>
```

Line items then add `<Valuty>$item["price"]</Valuty>` with the foreign-currency unit price.
